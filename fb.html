Disclaimer

I don't work for Facebook. This case study was a passion project of mine. It all started when I was trying to show my friend a meme on Facebook but had trouble finding it. The existing design is good as it allows users to categorize their saved posts to different categories. I started this case study as a passion project to challenge myself.

Goal

In this case study on Facebook's "Save Post" feature, I aim to improve the process of saving posts, creating collections, and finding saved posts, which could increase user engagement and clicking rates.

Understanding Facebook

Facebook is an application that is being used by 1.59 billion users worldwide daily (https://newsroom.fb.com/company-info/). Countless information is being viewed, read, and shared on the platform.

From CNN News, to travel guidelines, food recommendations, friend's life updates, funny videos, and relatable memes, Facebook is being used for everything. While absorbing information, at times, we like it, at times we share it, and at times we want to save it for later.

For this, there is the neglected "Save Post" feature. People save posts in many scenarios:when we find the information intriguing or sometimes when we find the post useful. One thing for certain is that people save posts because they want to be able to reference them in the future. However, people either do not know about the "Save Post" function or find it time consuming to find the post they have saved.

How does "Saved Post" work?

1. Allows users to save posts
The "Save Post" function allows users to save links, pages, posts, locations, etc.

2. Remind users about saved posts to view later
Facebook also reminds users about what items they have saved by sending out notifications.

So what's wrong?

I first found the problem when I was looking for a post I saved. I had trouble finding it. After the struggle, I wanted to redesign the feature and here it goes.

Key Findings

1. Low Discoverability to "Save Post"

The first problem occurs when a user is saving a post, it is difficult to find the saving button. In the current design, users have to click on the three dots on the top right corner, and select "save post". It is easy to overlook the "save post" button since it is hidden. Some users had never used it because they do not know it exists. According to the survey I sent out to 18 Facebook users, the average score for "process of saving a post" is 2.8 out of 10.

2. No Guidance/Customization

The second problem is that there is no guidance provided and customization availability when creating a new collection. Additionally, I found that the thumbnail photo is random, not scaled, and does not represent the collection in any way. (The default picture is a picture of a bird picking up a ring).

3. Hard to Find Saved Information

The third problem happens when the user is looking for a previously saved post, it is hard to find the saved information.¬†According to the online survey, 18 users' average "rating for process of saving a post" was 5/10.

Solution

I chose Design A because most users are already used to reaching to the top right corner to find the save button. The mental model has already been established; Therefore, it is more intuitive to keep it as is. Additionally, Design B is too cluttered, which would be difficult to use on mobile.

Problem 2: No Guidance/Customization

Redesign: Design A vs. Design B

Both designs aim to guide users with pre-existing options. It saves time for users on deciding collection name inputting collection name. This is important because the less time the user spent the more likely they would be willing to this function in the future. No users like to be interrupted when reading posts.

Solution

Comparing between Design A (list view) and Design B (grid view), I chose Design B (grid view) because there are more visual aids (icons), and users attention tends to be spread more evenly between each grid cell. Grid view is optimized for visual comprehension.
